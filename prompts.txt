To the same file add a function parse_line which takes a string and performs these operations:
- split the string by slashes into slash_parts
- take the first slash_part and split it by spaces, take the second of these parts as "hanzi"
- run the hanzi string through the pinyin function of the pinyin-pro library with option toneType set to 'none', apply the normalize function to the result, and call that result "npinyin"
- create a list called searchables with npinyin as its first element
- create an empty list called translations
- for each remaining slash_part:
  - append it to the translations list
  - apply the normalize function and append the result to the searchables list
- return a tuple consisting of hanzi, searchables, and translations
